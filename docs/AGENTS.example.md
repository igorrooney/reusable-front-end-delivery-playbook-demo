# Front-End Delivery Playbook Instructions

When working on front-end code in this repository, read and follow `docs/frontend-delivery-playbook.md`.

Use the playbook as reusable guidance for:

- Component structure
- Accessibility checks
- Form validation and error handling
- API loading, empty, error, and success states
- Front-end review quality
- Handover readiness

AI may use the playbook to suggest, draft, or review changes, but AI is a helper, not an authority. Technical leads and accountable engineers remain responsible for final validation and quality.

Do not add client names, client IP, confidential implementation detail, or assignment-specific code to reusable playbook guidance.

If a requested change conflicts with the playbook, explain the conflict and ask for confirmation before proceeding.
